Run HelloWorld using Gradle

Similar to Maven, we can declare JavaFX jars as dependency in the build.gradle file. However, for Gradle we need to find and specify the platform/OS as classifier. This requires a small script: 


buildscript {
    repositories {
        mavenCentral()
    }
    dependencies {
        classpath 'com.google.gradle:osdetector-gradle-plugin:1.6.0'
    }
}

apply plugin: 'application'
apply plugin: 'com.google.osdetector'

ext.platform = osdetector.os == 'osx' ? 'mac' : osdetector.os == 'windows' ? 'win' : osdetector.os

Next, we add all the dependencies using the platform. For instance, if we only need the javafx.controls module, we will include: 


dependencies {
    compile "org.openjfx:javafx-base:11:$platform"
    compile "org.openjfx:javafx-graphics:11:$platform"
    compile "org.openjfx:javafx-controls:11:$platform"
}
Note: The classifiers are not taken into account when resolving transitive dependencies in Gradle. Therefore, we need to specify all this three JavaFX modules with platform as classifier.

Next, we set --module-path to the value that would have been the classpath and add javafx.controls as a module to both javac and java commands. This makes sense because all the dependencies are currently on the classpath. 


compileJava {
    doFirst {
        options.compilerArgs = [
                '--module-path', classpath.asPath,
                '--add-modules', 'javafx.controls'
        ]
    }
}

run {
    doFirst {
        jvmArgs = [
                '--module-path', classpath.asPath,
                '--add-modules', 'javafx.controls'
        ]
    }
}
